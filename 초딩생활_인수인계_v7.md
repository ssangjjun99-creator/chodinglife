## 🎯 앱 개요: 초딩생활

**목적**: 초등학생 일과관리 + 부모-자녀 실시간 연동 앱
**타겟**: 초2~초5 학부모 (특히 초등 여학생)
**앱 이름**: 지율이가 직접 지어준 이름 (특별한 의미!)
**핵심 교감**: 아이 도착체크 → 엄마 확인 / 숙제 사진 올리기 → 엄마 확인
**수익모델**: 아이 1명 무료 / 2명째부터 유료
**참고 서비스**: 클래스카드 (선생님 11만명, 학생 230만명)
**차별점**: 파이차트 시각화 + 아이 직접 참여 + 숙제 사진 관리 + 부모↔아이 실시간 연동

## 🔥 비전 & 목표

```
지금 → Firebase 연동 마무리
↓
와이프 + 지율이 실사용 테스트
↓
React + Claude Code 전환 (디자인 보완)
↓
Capacitor 앱화
↓
구글 플레이스토어 출시 (안드로이드 먼저)
↓
애플 앱스토어 출시
↓
수익 창출 🎉
```

**경쟁앱 분석:**
- 이코딩: 학원→부모 단방향
- 하원나우: 부모 전용
- 키즈노트: 유치원용
- 위클링: 부모 전용, 리뷰 4개
- **초딩생활**: 아이 직접 참여 + 부모↔아이 양방향 + 파이차트 → 블루오션!

## 📱 앱 구조 (5개 화면)

### 1. 🌟 홈 (메인)
- 디지털 시계 왼쪽 위
- **파스텔 파이차트** (꽉 찬 파이, 오전/오후 분리)
- 시침만 있음 (분침 제거)
- **가운데 아이 사진** (부모설정에서 등록, Firestore 저장)
- 현재 섹터 주황 테두리 강조
- 이모지 + 이름 섹터 안에 표시

### 2. ⭐ 포인트
- 포인트 현황 헤더 (금주 누계 + 금일 점수)
- 오늘 도착 이력
- 이번주 숙제 완료 이력 ← **Firestore 연동 완료**
- 엄마 보너스 포인트 이력
- 이번주 보상

### 3. 📍 도착체크
- 스케쥴 기반 자동 추출
- 시작 30분 전 ~ 종료 30분 후 활성화
- +10점 즉시 지급

### 4. 📚 숙제완료
- **부모**: 부모탭만 보임 (숙제 완료 처리, 사진 확인)
- **아이**: 아이탭만 보임 (확인요청, 사진 첨부, 메모)
- 사진 Firebase Storage 양방향 연동

### 5. 👨‍👩‍👧 부모님
- 현황 탭: 포인트 + 보너스 버튼
- 스케쥴 탭: 위클리 그리드 (핵심!)
- 설정 탭: 아이 사진 등록, 아이랑 연동하기, 역할 초기화

## 🔑 Firebase 연동 현황

### Firebase 프로젝트
- **프로젝트명**: chodinglife
- **Firestore**: asia-northeast3 (서울)
- **플랜**: Blaze (유료, 무료 구간 충분)
- **Authentication**: 이메일/비번 + Google 로그인

### Firestore 데이터 구조
```
families/
  {familyCode}/
    data/
      schedule: { SCH, FREE, updatedAt }
      homework: { hwData, updatedAt }
      scores: { weekPts, todayPts, totalPts, updatedAt }
      hwlog: { hwLog, updatedAt }  ← 아직 연동 미완성!
      hw_photo_urls: { urls, updatedAt }
      profile: { childPhotoUrl, updatedAt }

users/
  {uid}/
    familyCode, email, displayName, createdAt
```

### Storage 구조
```
families/{familyCode}/
  hw_photos/{key}_{ds}.jpg  ← 숙제 사진
  child_profile.jpg          ← 아이 프로필 사진
```

## 🚨 절대 주의사항 (뻑난 경험 있음!)

### 1. 파이차트 각도 공식 (절대 건드리지 말 것!)
```javascript
relMinToAngle(relMin)  // 이 함수 하나만 사용
buildSegments()        // sentinel 객체 참조 동일성 중요
```
- 잘못 건드리면 홈화면 전체 망가짐 (실제 경험)

### 2. 스크롤 CSS (절대 건드리지 말 것!)
```css
.page: position:fixed; overflow-y:auto; -webkit-overflow-scrolling:touch
.sp: padding-bottom:80px
```

### 3. 사진 첨부 방식
- `<label>` + `<input type="file">` 방식 절대 금지 (overlay 충돌)
- 반드시 `triggerParentPhoto()` / `triggerChildPhoto()` 방식 사용
- 전역 hidden input을 body에 붙이고 `.click()` 으로 트리거

### 4. 파이차트 캔버스
- `pieC` 하나로 모든 것 그림
- `tick()` → `drawPie()` 호출 (drawH() 직접 호출 금지)
- clearRect는 `drawPie()` 시작에서만

### 5. 코드 수정 시
- 반드시 수정 전 파일 백업
- 문법 체크: `node --check` 로 JS 문법 검증
